Collabify - What to do after restarting your laptop
===================================================

1. Open Docker Desktop.
   - Wait until Docker says the engine is running.
   - If Docker asks to enable WSL 2 integration, allow it.

2. Run the continue batch file:

   scripts\windows\continue-after-restart.bat

   You can double-click it from File Explorer, or run it from PowerShell/CMD:

   cd /d D:\DevProjecto\collabify-v2
   scripts\windows\continue-after-restart.bat

3. What the batch file does:
   - checks GitHub CLI
   - checks Node/npm
   - updates WSL
   - sets WSL default version to 2
   - checks Docker
   - checks Supabase CLI via npx
   - runs local Supabase migration validation:
     npx supabase db reset --debug
   - runs:
     git diff --check

4. If it succeeds:
   Send the success message back here.
   Next recommended task will be RLS seed/test automation, then frontend auth onboarding.

5. If it fails:
   Copy the full error output and send it back here.

6. If Docker still says Windows Hypervisor is not present:
   Open PowerShell as Administrator and run:

   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   dism.exe /online /enable-feature /featurename:HypervisorPlatform /all /norestart
   bcdedit /set hypervisorlaunchtype auto

   If you are on Windows Pro or Enterprise, also run:

   dism.exe /online /enable-feature /featurename:Microsoft-Hyper-V-All /all /norestart

   Then restart again.

7. Important:
   Do not put Supabase service role keys in the React frontend.
   Frontend should only use:
   - VITE_SUPABASE_URL
   - VITE_SUPABASE_ANON_KEY
